# Bruno — Reunião com cliente Aurora Pay

## Objetivo do exemplo

Mostrar como preparar uma reunião com cliente usando IA sem cair em uma pauta genérica.

O foco é transformar dados soltos sobre contexto, problemas, riscos e dúvidas em uma pauta de 30 minutos com decisões esperadas e próximos passos.

## Persona

Bruno trabalha em um time de produto/tecnologia e precisa se preparar para uma reunião com a Aurora Pay, cliente fictício que reportou dúvidas sobre conciliação e divergências operacionais.

## Dor principal

A reunião pode virar uma conversa solta, sem decisão, se Bruno não chegar com objetivo, perguntas certas, riscos e critérios de sucesso.

## Arquivos da pasta `dados/`

| Arquivo | Para que serve |
|---|---|
| `contexto.md` | Briefing do cliente fictício e objetivo da reunião |
| `sinais-e-riscos.md` | Pontos de atenção que precisam virar perguntas |
| `amostra-dados.md` | Amostra simplificada dos dados mock de conciliação |

## Aprendizado central

Para trabalho, uma boa conversa com IA não é só pedir uma pauta. É dar contexto suficiente para que a pauta termine em decisão, responsáveis e próximos passos.
